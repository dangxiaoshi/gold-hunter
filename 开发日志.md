# 金币猎人 V2 开发日志

## 2026-04-05

### 对话框通讯录与客户数据持久化

- 调整 `server.js` 的客户同步逻辑，打开 `/api/customers` 时会自动检查微信聊天源文件是否变化。
- 当聊天源文件有更新时，服务端会自动重载缓存，并把新联系人补进 `data/customers.json`，不再依赖手动点击“导入聊天记录”。
- 手动新增客户后，前端会自动清空搜索条件、把该客户插到列表顶部，并立即选中打开，避免“新增后找不到人”。
- 补充客户额外消息持久化接口：手动粘贴到对话框中的聊天内容现在会写回服务端，而不是只放在浏览器 `localStorage`。
- 已兼容旧的本地 `gh_extra_msgs_*` 缓存迁移；重新打开客户时会尝试把旧缓存补写回服务端。

### 出击与参考原文交互优化

- 对话框中的“参考原文”改为默认折叠，只显示标题信息。
- 点击标题后再展开正文，再点一次可收起，减少 AI 区域信息堆叠。
- 原先“出击 + V1切换”改为单一入口：只保留 `⚡ 直接出击`。
- 移除旧版出击按钮和 `V1` 开关，点击后直接走原先的 `V1` 话术生成逻辑，降低决策成本。

### 客户列表操作菜单

- 左侧客户卡片新增上下文菜单，支持 `置顶 / 标为未读 / 不显示 / 删除`。
- 菜单最初做成双击触发，后续根据操作迁移成本优化为更接近微信习惯的 `右键 / 触控板双指点按`。
- 置顶客户会显示 `📌` 并在列表排序时优先展示。
- 标为未读的客户会显示红点，刷新后仍保留。
- “不显示”会把客户标记为隐藏；“删除”会删除客户资料和手动补充消息。
- 上述状态均已写入服务端，刷新页面后仍可保持。

### 运行与测试

## 2026-04-04

### 知识库切块重构

- 调整 `server.js` 中的 `chunkDoc()`，不再按“`【MM.DD】` 一天一个块”直接入库。
- 新逻辑改为两级切块：先按 `【MM.DD】` 切出每天，再按正文中的 `👇` 话题分隔切出独立块。
- 新块结构统一为：`docTitle / date / topic / tags / content`。
- `topic` 取 `👇` 后面紧跟的那一行文字，去掉 `👇` 前缀和首尾空格。
- `tags` 按 `topic` 中的 `+` 拆分生成，便于后续检索加权。
- 如果某天没有 `👇` 分隔，则整天内容 fallback 为一个块。
- 补了一层正文清洗，避免 `content` 开头残留与 `topic` 重复的 `👇 标题行`。
