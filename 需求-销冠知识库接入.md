## 一、背景

### 项目背景
「金币猎人」是一个 Web 销售辅助工具（`/Users/dang/金币猎人2`，Node.js + Express）。
销售人员在工具里看客户聊天记录，点「出击」按钮，AI 生成话术建议。

### 现有问题
AI 生成的话术**太通用、太笨**，不够口语化，不够贴近实际销售场景。
现在的 AI 只知道聊天记录，不知道经过验证的销售方法论和真实话术风格。
→ 话术拿出来还要自己大改，等于没省脑力，工具就没有意义。

### 核心目标
让 AI 生成的话术**直接能发，或者微调就能发**。
不是替代思考，是复刻经验——AI 要懂我的打法、我的腔调、我的方法论。

## 三、核心方案

### 设计原则
> **不改原文，只打标签。**  
> AI 读原文生成话术，风格和腔调来自这些群聊原文，不是来自提炼版本。

### 数据处理流程

```
飞书知识库（76篇月度文档）
    ↓ 先按【MM.DD】切出每天
    ↓ 再按标题里的「+」切出每个话题段落
每个话题段落 = 一个独立块
    ↓ 标题关键词直接作为基础标签
    ↓ AI 读段落内容，补充隐含标签
结构化存储（原文不动，只加标签）
    {
      "date": "01.01",
      "topic": "嫌价格贵谈单案例",
      "tags": ["价格异议", "嫌贵", "谈单"],
      "content": "...原始群聊原文，完整保留..."
    }
```

### 出击时的检索流程

```
销售输入客户场景（或自动从聊天记录提取）
    ↓
关键词/标签匹配 → 找到最相关的 3-5 个日期块
    ↓
把这些块的原文注入 Claude 的 system prompt
    ↓
Claude 生成话术（参考原文腔调，不改写）
    ↓
输出：话术建议 + 参考来源（哪天的精华）
```

### 对用户透明
点同一个「出击」按钮，背后多两步，慢 1-2 秒，话术质量大不同。

## 五、已确认决策

### 切块粒度：按段落 ✅
- 同一天内不同话题拆成独立的块
- 切块依据：每天标题里的 `+` 分隔符（如「为什么客户说先观望观望**+**嫌价格贵谈单案例」）
- 一个话题 = 一个块，检索更精准，不会多个主题混在一起

### 打标签方式：标题关键词 + AI 补充 ✅
- 先把每段小标题的关键词直接作为基础标签（快）
- 再让 AI 读段落内容，补上标题没覆盖到的隐含标签（准）
- 省约一半 API 调用

### 更新后的存储格式
```json
{
  "updatedAt": "2026-04-04T12:00:00Z",
  "totalChunks": 800,
```
