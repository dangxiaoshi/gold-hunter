### HTML 结构说明（重要）

标签**不在 HTML 属性里**，而是写在正文 `<p>` 标签内：

```html
<div class="memo">
  <div class="time">2025-12-02 16:24:17</div>
  <div class="content">
    <p>#💰/销售 </p>
    <p>虽然对方说自己不在乎名利，但你也不要拆穿对方...</p>
  </div>
</div>
```

解析时需从文字内容里提取 `#标签`，不能从 HTML 属性读。

## 技术方案

### 新增文件
- `data/knowledge_base.json` — 解析后的知识库，一次性脚本生成
- `scripts/parse_flomo.js` — 解析脚本（单独运行，不在服务启动时执行）

### 新增 config.json 配置项

```json
{
  "flomoPath": "/Users/dang/Downloads/flomo@dangxiaoshi-20260404/dangxiaoshi的笔记.html"
}
```

路径变动时只改这一处。

---

### 解析脚本（`scripts/parse_flomo.js`，一次性手动运行）

1. 读取 HTML 文件
2. 提取每条 `.memo` 的：时间、正文内容
3. 从正文里用正则提取 `#tag` 标签
4. 只保留含 `💰/销售`、`💰/金句`、`💰` 标签的条目
5. 去掉正文里的标签行，只保留实际内容
6. 存入 `data/knowledge_base.json`

```json
{
  "updatedAt": "2026-04-04",
  "totalChunks": 322,
  "chunks": [
    {
      "time": "2025-12-02 16:24:17",
      "tags": ["💰/销售"],
      "content": "虽然对方说自己不在乎名利，但你也不要拆穿对方..."
    }
  ]
}
```

---

### 检索逻辑（出击时调用）

原方案是纯关键词匹配，但问题是：聊天里客户说的是口语（"我想想""有点贵"），知识库里是技巧总结，两边词汇空间不一样，容易匹配不上。

**改进方案：阶段优先 + 关键词补充**

```
输入：当前销售阶段 + 聊天关键词
  ↓
第一步：按阶段过滤（优先取与当前阶段匹配的语料）
  阶段映射：
  - 建立链接 / 同步信息  → 优先取「建立信任、破冰」相关
  - 挖掘需求            → 优先取「挖痛点、封闭选项」相关
  - 解决顾虑            → 优先取「价格异议、时间顾虑、效果质疑」相关
  - 达成成交            → 优先取「临门一脚、紧迫感」相关
  ↓
第二步：在阶段匹配结果里，再用关键词排序
  关键词从最近 10 条聊天里提取（价格/贵/时间/效果/考虑/担心/成功案例...）
  ↓
取最相关的 5 条
  ↓
注入 system prompt
```

---

### 修改出击接口

在 `POST /api/ai/tactics-v1`（以及 `/api/ai/tactics`）里：

1. 加载 `data/knowledge_base.json`（**若文件不存在则跳过，不报错**）
2. 用上述检索逻辑取 3-5 条
3. 在 system prompt 末尾加入：

```
--- 参考话术语料（这是真实销售中说过的原话，照这个腔调来，不要改写）---
1. ...
2. ...
3. ...
```

**前端不动，用户无感知，只是话术质量变好。**
